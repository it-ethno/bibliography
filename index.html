String Formats

Main Input

There are certain formats of input you can utilize with the big main input interface. If you enter the information of a book/article/etc. in one of the defined formats, your new title will automatically be created without the need to fill in any form.
The current formats defined are as follows:

Book

<Surname Author>, <Prename Author>. <Year of Publication>. <Title>: <Subtitle (optional!)>. <Place of Publication>: <Publishing House>.

Article in Book

<Surname Author>, <Prename Author>. <Year of Publication>. <Title>: <Subtitle (optional!)>. in: <Surname Editor>, <Prename Editor>. <Title of the containing book>. <Place of Publication>: <Publishing House>.

Article in Journal

<Surname Author>, <Prename Author>. <Year of Publication>. <Title>: <Subtitle (optional!)>. in: <Title of the Journal>, <Issue Nr. or Date>. <Place of Publication>: <Publishing House>.
When you are a scientist or use your bibliographic collection in any academic work, you surely want to configure these strings to fit your institutional requirements.

Right now (end of August 2022) the PDF export is still a little bit buggy and produces unwished results when the exported titles exceed one single page. Furthermore the exported files are rather large and text inside them can not be selected.
The development plan for Bibliography provides for the following improvements:
End of October 2022: PDF-Export utilizing node.js technologies, shrinking file sizes and enabling text selection.
End of 2022: More technology options for PDF-Export.

Storage

Right now (end of August 2022) the only working option for storage is the Local Storage of your browser, which is limited to something between 5 to 10 mb. This is still enough for several hundred books and quotes, but the data should be backed up to a file regularly, because the Local Storage is subject to the browsers own memory eviction policies and could be deleted without notice.
The development plan for Bibliography provides for the extension of the storage options as follows:
End of September 2022: Storage of data inside the IndexedDB of the browser, which will increase the available memory to something between some hundred mb up to several gigabyte, depending on the platform and browser used.
End of October 2022: Storage of data inside concrete files on users hard disk utilizing node.js functionality. This feature will only be available from inside the installable app package of Bibliography.
End of 2022: Network Storage of data on a special server designed to work in library settings as well as home server.
